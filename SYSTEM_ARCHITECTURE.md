## JWT Token Structure

```
JWT Token Structure:
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

Header.Payload.Signature

Example Decoded Token:
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "id": 123,                          в—„в”Җв”Җ User ID
    "email": "user@example.com",        в—„в”Җв”Җ User Email
    "iat": 1646899200,                  в—„в”Җв”Җ Issued at
    "exp": 1647503000                   в—„в”Җв”Җ Expires at (7 days)
  },
  "signature": "base64encodedstring"
}

How it's used:
1. Generated on login/register
2. Stored in localStorage as intelToken
3. Sent in every API request header:
   Authorization: Bearer {token}
4. Verified on backend using JWT_SECRET
5. Auto-logout when expired (401 response)
```
